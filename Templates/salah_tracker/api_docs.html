React Native API Documentation

Complete guide to integrate the Django Backend with your Expo Application. Base URL: http://<YOUR_IP>:8000

Note on Authentication: There is currently no strict token authentication implemented. Every request expects a username field to be passed identifying the current logged-in user in your Expo app.
GET /api/daily-prayers/?username={username}

Fetches today's Fard and Sunnah prayers along with their completion status for the specific user.

Query Parameters
  • username (string, required): The username of the logged-in user.
Success Response Example
{
  "status": "success",
  "date": "2026-04-13",
  "fards": [
    { "id": 1, "name": "Fajr", "completed": false, "completed_time": null }
  ],
  "sunnahs": [
    { "id": 1, "name": "2 Rakat Before Fajr", "fard_link": "Fajr", "completed": false, "completed_time": null }
  ]
}
POST /api/toggle-fard/

Toggles the completion status (True/False) of a specific Fard prayer.

JSON Body
{
  "username": "johndoe",
  "id": 1
}
Success Response Example
{
  "status": "success",
  "completed": true
}
POST /api/toggle-sunnah/

Toggles the completion status (True/False) of a specific Sunnah prayer.

JSON Body
{
  "username": "johndoe",
  "id": 1
}
Success Response Example
{
  "status": "success",
  "completed": true
}
POST /api/add-azkaar/

Logs a new Zikr count.

JSON Body
{
  "username": "johndoe",
  "name": "Subhanallah",
  "count": 33
}
Success Response Example
{
  "status": "success",
  "id": 5,
  "name": "Subhanallah"
}
POST /api/add-quran/

Logs new Quran tracking stats.

JSON Body
{
  "username": "johndoe",
  "surah": "Al-Baqarah",
  "start_ayah": 1,
  "end_ayah": 5,
  "notes": "Beautiful recitation"
}
Success Response Example
{
  "status": "success",
  "id": 2,
  "surah": "Al-Baqarah"
}
POST /api/add-nafl/

Adds a recorded Nafl prayer timestamp automatically.

JSON Body
{
  "username": "johndoe",
  "name": "Tahajjud"
}
Success Response Example
{
  "status": "success",
  "id": 3,
  "name": "Tahajjud"
}
GET /api/stats/?username={username}&days={days}

Fetches extreme detailed analytics correctly serialized.

Query Parameters
  • username (string, required): The username of the logged-in user.
  • days (int, optional): The number of days to analyze (e.g. 30). Defaults to 30.
Success Response Example
{
  "status": "success",
  "summary": {
    "fard_completed_today": 3,
    "fard_total_today": 5
  },
  "detailed_stats": {
    "prayer_stats": { ... },
    "quran_stats": { ... },
    "azkaar_stats": { ... },
    "nafl_stats": { ... }
  }
}
GET /api/next-prayer/?username={username}

Fetches the next upcoming prayer based on current time and the user's saved prayer times.

Query Parameters
  • username (string, required): The username of the logged-in user.
Success Response Example
{
  "status": "success",
  "next_prayer": "Asr",
  "time": "15:30:00",
  "is_tomorrow": false
}
